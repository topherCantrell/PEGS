Variable Operations

CCL has 128 variables you can use in your program. You can reference them by number as “V0” through “V127”, but you will usually assign them names like “numberOfLives” so you can remember what each variable holds.

 

Variables are 4 byte unsigned numbers. A variable can hold any value from 0 to 4,294,967,295.

 

In CCL all variable expressions are built from five basic fields in this order:

Destination, Process, Left, Operation, Right

 

Consider the following example:

V100 = V120 + 278

 

The “destination” field is the far left term. This is where the result ends up in an assignment process. In this example, the value of “V120+278” is stored in the destination “V100”.

 

The “process” field describes what happens to information in the expression. In this example, the process is a simple assignment. The result of the right side of the process is stored in the destination. There are other processes that compare the left and right side and return either a 1 or a 0 (see below). Comparison processes are used in IF statements.

 

The right side of the process field is a math expression with a “left” field, an “operation” field, and a “right” field. In the example, “V120” is the left, “+” is the operation, and “278” is the right field.

 

Some operations ignore the left field completely. The RND (random number) operation returns a 32 bit random number that is ANDed with the right field. The left field is ignored. The bitwise NOT operation returns the 1’s compliment of the right field completely ignoring the left field.

 

The left field and operation field are optional. In many assignment examples, only the right field is needed (see below). In many IF expressions, the left and process is optional. The right side of the variable expression is used for decisions.

 

The three value fields (destination, left, and right) can be one of three things: constant, variable, or a pointer. Constants are plain numbers coded directly into the expression. Variables are the value in a single variable. Pointers use one variable to point to another. For instance, if V1 had the value 100 and V2 had the value 1, then “[V2]” would resolve to “100”. In this case, V2 points to V1 which holds 100.

 

Destination, Left, and Right:

Vn                    The value is taken directly from the variable V0 through V127

[Vn]                 A variable points to another variable whose value is used.

1234                The value is a numeric constant

 

Process:

=                      Assign the result on the right to the variable on the left. Return result.

= =                   Return 1 if result on the right is the same as variable on the left.

!=                     Return 1 if result on the right is different from variable on the left.

<                      Return 1 if variable on left is less than result on right.

<=                    Return 1 if variable on left is less than or equal to result on right.

>                      Return 1 if variable on left is greater than result on right.

>=                    Return 1 if variable on left is greater than or equal to result on right.

NONE             Do nothing. Return result on right.

 

Operation:

+                      Result is left term added to right.

-                       Result is left term minus right.

*                      Result is left term multiplied by right.

/                       Result is left term divided by right.

%                     Result is left term modulo right

<<                    Result is left term shifted left right term times.

>>                    Result is left term shifted right right term times.

&                     Result is left term ANDed with right term.

|                       Result is left term ORed with right term.

^                      Result is left term XORed with right term.

~                      Result is NOT of right term (left term ignored).

RND                Result is random 32 bit value ANDed with right term (left term ignored).

NONE             Result is right term (left term ignored).
