Text Commands

CLS      

Clear the screen

This command fills the entire screen with the SPACE character and resets the text cursor to the top left corner of the screen.

 

CLS // Clear the screen

 

CLS n   

Clear the screen with a specific character

This command fills the entire screen with the given character and resets the text cursor to the top left corner of the screen.

 

CLS "." // Fill the screen with periods

CLS "#" // Fill the screen with pounds

 

PRINT “message”     

Print a given message

This command prints a message on the screen at the current text cursor.

 

PRINT "HELLO WORLD\n"

PRINT "TESTING"

 

PRINT label       

Print a message defined in the data section

This command prints a message on the screen at the current text cursor. The message must be defined in the data section.

 

PRINT myMsg

 

-- DATA --

 

myMsg:

"HELLO WORLD\n", 0

 

PRINTVAR Vn         

Print a number

This command prints the value of the given variable on the screen at the current text cursor.

 

PRINTVAR V0   // Print the contents of V0

PRINTVAR V32  // Print the contents of V32

 

GETNUMBER Vn           

Read a number from the keyboard

This command reads a numeric value from the user on the screen at the current text cursor. Use the “SETCURSORINFO” command to change the picture and blink-rate of the cursor.

 

GETNUMBER V0  // Get a number from the user

 

GETLINE label, size 

Read a length of text from the keyboard

This command reads a line of text from the user on the screen at the current text cursor. The size is the maximum number of characters to accept. Use the “SETCURSORINFO” command to change the picture and blink-rate of the cursor.

 

Note that the text is automatically zero-terminated after reading. Make sure the buffer is large enough the extra zero on the end.

 

If the “label” does not already exist in the data section, a data line is automatically created and sized.

 

GETLINE myText, 20  // Get 20 characters from the user

 

TOKENIZE buf,ptr,Vn,maxWords

       Convert an input line into a series of numeric tokens

This command tokenizes a line of text (user input) and fills a sequence of variables with the numeric tokens mapped to each word.

 

Token information must be defined in the data section as a list of zero-terminated strings followed by the 2-byte numeric value assigned to the string. The list of tokens is terminated with a zero string (see example below).

 

The first variable is filled with the number of words (separated by spaces) on the input line. The second variable is filled with the number of words that are decoded. This will be less than or equal to “maxWords”.

 

The following variables are filled with the numeric values of the decoded words or 0xFFFFFFFF if the word has no match in the token info table.

 

In the example below, the variables will be filled as follows:

V10 = 6 (There are six words in the input)

V11 = 4 (As requested, a max of four words were decoded)

V12 = 0x1000

V13 = 0x2000

V14 = 0xFFFFFFFF (The work “FOUR” is not in the token table)

V15 = 0x3000

 

TOKENIZE inp, tokenInfo, V10, 4

 

-- DATA --

 

inp:

"ONE      TWO FOUR THREE A B" , 0

 

tokenInfo:

"ONE", 0,    word(0x1000)

"TWO", 0,    word(0x2000)

"THREE", 0,  word(0x3000)

0

Tile Commands

INITTILES ptr, start, count

       Initialize a sequence of graphic tiles

This command changes tile-picture-memory for one or more tiles defined in the data section at label “ptr”. The “start” indicates the starting tile-number to change, and count indicates the number of tiles defined.

 

Tiles are drawn in ASCII art as shown below.

 

INITTILES pics, 10, 3  // Init tiles 10, 11, and 12

 

-- DATA --

 

" | W......R  GGGGGGGG  RRRRRRRR"

" | .W.....R  G......G  ......R."

" | ..W.....  G......G  .....R.."

" | ...W....  G......G  ....R..."

" | ....W...  G......G  ...R...."

" | .....W..  G......G  ..R....."

" | G.....W.  G......G  .R......"

" | G......W  GGGGGGGG  R......."
